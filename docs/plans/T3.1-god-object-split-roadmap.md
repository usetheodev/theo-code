## Why this is its own document

The remediation plan classified T3.1 as `L` (large). The single
`AgentRunEngine` struct has:

- 44 fields in `crates/theo-agent-runtime/src/run_engine/mod.rs:50-136`
- field references from 23+ intra-crate modules
- ~50 test fixtures that build the engine via field-by-field literals

A big-bang rewrite would touch 50+ files in one PR with high regression
risk for the 1216 lib tests. This roadmap breaks the work into 5
incremental PRs (one struct-extraction per PR) so each PR stays
bisect-friendly and the test suite stays green at every step.

## PR sequence

### PR1 — `SubagentContext` (~8 fields)

The `subagent_*` field cluster is the **most natural first slice** —
all 8 fields share a prefix and a single conceptual responsibility
(sub-agent integration plumbing).

**Fields to move:**

```
subagent_registry          : Option<Arc<SubAgentRegistry>>
subagent_run_store         : Option<Arc<FileSubagentRunStore>>
subagent_hooks             : Option<Arc<HookManager>>
subagent_cancellation      : Option<Arc<CancellationTree>>
subagent_checkpoint        : Option<Arc<CheckpointManager>>
subagent_worktree          : Option<Arc<WorktreeProvider>>
subagent_mcp               : Option<Arc<McpRegistry>>
subagent_mcp_discovery     : Option<Arc<DiscoveryCache>>
subagent_handoff_guardrails: Option<Arc<GuardrailChain>>
subagent_reloadable        : Option<ReloadableRegistry>
```

**File:** `crates/theo-agent-runtime/src/run_engine/contexts/subagent.rs` (NEW).

**Migration steps:**
1. Define `pub(crate) struct SubagentContext { … }` with the 10 fields above.
2. Replace the 10 `AgentRunEngine` fields with a single `subagent: SubagentContext`.
3. Add forwarding accessors on `AgentRunEngine` (`fn subagent(&self) → &SubagentContext`).
4. Update `delegate_handler.rs` and other call sites to read from
   `self.subagent.<field>` instead of `self.subagent_<field>`.
5. Update test fixtures (~50 files in `subagent/mod.rs` tests + worktree_override module) by introducing `SubagentContext::default()` / a test-builder.

**Estimated diff:** ~500 LOC across ~15 files.

dead post-extraction.

After PR5 the `AgentRunEngine` struct should contain ≤ 8 top-level
fields (5 contexts + `config: AgentConfig` + `project_dir: PathBuf`
+ `task_manager: Arc<TaskManager>`).

## Why we did NOT land this in the Ralph loop

The Ralph loop deferred T3.1 because:

1. **Each PR is a bisect-friendly atomic change** — splitting them
   across iterations would leave the engine in a half-migrated state
   between commits. The 5-PR boundaries are the right granularity.
2. **Test fixtures need coordinated update** — many tests build
   `AgentRunEngine` field-by-field; converting them in batches keeps
   each PR self-contained. Mixing this with other unrelated cleanups
   (T4.10) muddles diff review.
3. **Plan explicitly tagged L (5+ PRs, multi-week)** — the plan
   itself anticipated this would not fit in a single iteration cycle.

The 29/31 cards completed by the Ralph loop establish the foundation
(unbroken CI gates, all hygiene tasks done) on which T3.1 can land
safely as a follow-up multi-PR effort.
