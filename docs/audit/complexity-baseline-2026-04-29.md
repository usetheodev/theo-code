## Path to Phase 4 completion (T4.2 .. T4.9)

The plan calls for driving each crate's ceiling to **0**. The current
posture is that every crate is at-or-below its allowlisted ceiling, so
gates pass but no work has been done to reduce the count.

T4.2..T4.9 would extract helpers from each long function, dropping the
count crate-by-crate. Each extraction needs:
- Characterization tests *before* extraction (per existing 5247-test net)
- Pure helpers (no side effects) preferred
- Per-task TDD discipline

Given the volume (74 functions) and risk (touching runtime-critical paths
like `execute_with_history` and `assemble_with_code`), this snapshot
defers the full decomposition to a follow-up effort. The current posture
is honest: complexity is documented, ceilings are tight (theo-tooling
refreshed to 7), no NEW violations can land without an allowlist update.
